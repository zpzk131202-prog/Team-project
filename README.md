# 개발 규칙

이 프로젝트는 **유튜브 시청자 참여형 게임 개발** 입니다.  
모든 기여는 GitHub 규칙을 따라야 하며, 관리자의 판단이 최종입니다.

---

## 1. 기본 원칙
- 모든 참여는 **Pull Request(PR)** 로만 받습니다.
- 관리자는 언제든 PR을 거절할 수 있습니다.
- 규칙 미준수 PR은 사전 예고 없이 Close 될 수 있습니다.

---

## 2. 참여 방법
1. 이 저장소를 **Fork** 합니다.
2. `develop` 브랜치 기준으로 **feature 브랜치**를 생성합니다.
3. 작업 후 `develop` 브랜치로 **Pull Request**를 생성합니다.

❌ `main`, `develop` 브랜치에 직접 push 금지

---

## 3. 브랜치 규칙
- `main` : 배포 및 방송용 (관리자 전용)
- `develop` : 시청자 PR 병합 브랜치
- `feature/*` : 개인 작업 브랜치

---

## 4. Pull Request(PR) 규칙 ⭐⭐⭐
아래 조건을 **모두 만족**해야 리뷰 대상이 됩니다.

- PR 설명 필수
- 변경 내용이 명확해야 함
- 게임이 정상 실행되어야 함
- 의미 없는 변경 금지  
(공백, 주석만 수정, 변수명만 변경 등)

❌ 규칙 위반 PR은 즉시 Close 됩니다.

---

## 5. 작업 허용 범위
### ✅ 허용
src/
assets/


### ❌ 금지


.github/
.gitignore
프로젝트 설정 파일
빌드 결과물


---

## 6. Issue 규칙
- **Issue 없는 PR은 받지 않습니다**
- PR 제목에 Issue 번호를 포함해야 합니다.


fix #12 플레이어 점프 구현


---

## 7. 금지 행동 🚫
- 욕설, 비방, 도배
- 악성 코드 삽입
- 고의적인 프로젝트 훼손
- 트롤 목적의 PR

위반 시 PR 거절 및 GitHub 차단될 수 있습니다.

---

## 8. 방송 중 운영 규칙
- 라이브 중에는 **Merge 하지 않습니다**
- PR 리뷰는 방송 콘텐츠로 사용될 수 있습니다
- GitHub 닉네임은 방송에 노출될 수 있습니다

---

## 9. 트롤 대응 정책
- 1~2회 위반: PR 거절 + 사유 안내
- 반복 위반: GitHub 사용자 차단
- 방송 중 공개 저격은 하지 않습니다

---

## 🔥 요약
> 규칙을 지키면 누구나 환영합니다.  
> 규칙을 어기면 조용히 참여가 제한됩니다.
